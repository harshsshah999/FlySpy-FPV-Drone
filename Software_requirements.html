Software Component Functional Requirements

Software Component Function / Purpose Requirement Justification
Flight Stack (Betaflight / PX4 / ArduPilot) Handles flight stabilization, motor control, telemetry. Betaflight is ideal for FPV/manual control; PX4 or ArduPilot is required if full autonomy and GPS-based autopilot are needed.
Ground Control Station (QGroundControl) Used for mission planning, telemetry monitoring, and parameter tuning. Essential if PX4 or ArduPilot is used; not needed for Betaflight-based manual flight configurations.
Operating System (Raspberry Pi OS / Ubuntu) Provides the software environment for running AI and control tasks on the SBC. Linux-based OS required for compatibility with most open-source AI and robotics frameworks.
AI Framework (e.g., YOLOv5, TensorFlow Lite) Performs object detection and inference tasks in real time. Chosen for lightweight and real-time performance on edge devices with AI accelerators.
Telemetry / Logging Software
